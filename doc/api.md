# API Design

## Stack: NestJS

**Why NestJS over plain Express/Fastify:**
- Module system enforces separation of concerns ‚Äî each feature is a self-contained module (controller, service, DTOs)
- Built-in dependency injection makes services easy to test and compose
- First-class TypeScript with decorators for validation, guards, and interceptors
- `@nestjs/websockets` for Socket.io is a native integration, not a bolt-on

---

## Conventions

### URL Structure
```
/api/<resource>           collection
/api/<resource>/:id       single item
/api/<resource>/me        current user's own resource (avoids exposing IDs in URLs)
```

Examples:
```
GET  /api/doctors               public doctor list
GET  /api/doctors/:id           public doctor profile
GET  /api/doctors/me            current doctor's own profile (doctor only)
GET  /api/patients/me           current patient's own profile (patient only)
GET  /api/appointments/mine     current user's appointments
```

### HTTP Methods
| Method | Use |
|---|---|
| `GET` | Read ‚Äî never mutates state |
| `POST` | Create a new resource |
| `PATCH` | Partial update ‚Äî only send changed fields |
| `DELETE` | Hard delete (used only for availability slots) |

`PUT` is not used ‚Äî `PATCH` is always preferred for partial updates.

### Response Shape
Success:
```json
{
  "data": { ... }
}
```

Error:
```json
{
  "statusCode": 404,
  "message": "Appointment not found",
  "error": "Not Found"
}
```

NestJS's built-in exception filters handle this format automatically via `HttpException` subclasses.

## DTOs and Validation

All request bodies are typed with `class-validator` + `class-transformer` DTOs. The global `ValidationPipe` is applied with `whitelist: true` (strips unknown fields) and `forbidNonWhitelisted: true`.

Types generated by Prisma are re-exported from `packages/types` so the frontend can use the same shapes without duplicating type definitions.

---

## Error Handling

- **400** ‚Äî validation failure (DTO mismatch)
- **401** ‚Äî missing or invalid JWT
- **403** ‚Äî wrong role for the endpoint
- **404** ‚Äî resource not found (via `NotFoundException`)
- **409** ‚Äî conflict (e.g., double-booking a slot)
- **500** ‚Äî unexpected errors caught by a global exception filter and logged

Business rule violations (e.g., booking a blocked slot) return **409 Conflict** with a descriptive message, not a generic 400.

---

## Pagination

List endpoints use cursor-based pagination:
```
GET /api/doctors?cursor=<lastId>&limit=20
```

Response includes `nextCursor` when more results exist. Offset pagination is avoided ‚Äî it performs poorly on large tables and produces inconsistent results when records are inserted between pages.
